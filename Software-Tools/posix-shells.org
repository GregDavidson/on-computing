* POSIX Shells

** Key Terms

- [[https://en.wikipedia.org/wiki/Shell_(computing)][Shell]] - operating-system user-interface
      - Can be Graphical or Text based
      - Graphical Shells are more casual
      - Text-based Shells are more powerful
- [[https://en.wikipedia.org/wiki/Command-line_interface]["Command-Line" Interface]] - text-based command interface
      - Commands are often expressed as a single line of text
      - Commands can also consist of multiple lines or be only part of a line!
      - The term "Command-Line Interface" or CLI is really a misnomer!
- [[https://en.wikipedia.org/wiki/POSIX][POSIX Operating System Standard]]

** POSIX-Compatible Shells

*** sh - Bourne shell and Bourne Compatible Shells

The [[https://en.wikipedia.org/wiki/Bourne_shell][Bourne shell]] written by Stephen Bourne appeared in /Unix Version 7/ in 1979,
replacing the original Unix shell written written by Ken Thompson. Both shells
were named ~sh~.

POSIX systems require a System-Shell which is compatible with the Bourne Shell
for essential operations, including logging into an interactive user account.

Many later shells are compatible with the command-language of the Bourne Shell.
