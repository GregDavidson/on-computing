Preface

This Simple Note presents a model computer system roughly comparable to a modern 32-bit personal computer, pared down to its essentials. This model computer is then used as a baseline for understanding a variety of actual computers, both common and exotic. Here are its specs (which will be fully explained by the end of this note):

  • 1 32-bit CPU core running at 2 GHz
  • Up to 4 Gi Bytes Dual-Ported ECC RAM
  • 1 Video subsystem driving a 2 Ki by 1 Ki wide-screen monitor, i.e. 2048 pixels wide by 1024 pixels high.
  • 8 24-bit 88 KHz audio channels
  • 4 1 GHz ethernet channels
  • 1 USB 2.0 bus
  • 2 500 Gi byte hard disk drives
  • 1 DVD reader/burner
  • 4 software-defined radios
  • A variety of USB equipment including a pointing device such as a mouse and a keyboard with extra USB ports. The keyboard can be plugged into the computer and any other USB devices can then be plugged into the keyboard.

To understand this computer we must understand

  1. How this computer uses its memory.
  2. How this computer uses its devices.
  3. What operations this computer can do.
  4. How the instructions which control this computer can be stored as data.
  5. How data, including instructions, can create abstractions which make a much better system.

Main Memory vs. Registers

Our model computer supports up to 4 Gi Bytes of main memory (RAM). That's 32 Gi bits or 34,359,738,368 ON/OFF switches. Amazing, isn't it? The switches in the main memory are wired together so that the CPU can load (read) or store (write) their contents by number. The first byte in main memory is byte 0, the next is byte 1, and so on all the way up to byte 4,294,967,295 which is the last of the 4 Gi Bytes of memory. The number indicates the location or address of the byte in main memory.

The computer's CPU has a much smaller number of switches ganged together in groups of 32. Each group of 32 switches in the CPU is called a register. The fact that the CPU's registers are all 32-bits wide is why this is a 32-bit CPU. Registers are much faster to read and write than main memory, often by a factor of 10 or more, so fast programs try to keep frequently used values in Registers rather than in main memory.

Most of the CPU's Registers can be used to hold any one-word value at the programmer's discretion - these are called the General Purpose Registers. A few of the registers have special dedicated purposes, here are a few of those:

PCR = Program Counter Register
Holds the location in main memory of the next instruction to perform. After most instructions the PCR is automatically incremented to address the following instruction.
FPR = Frame Pointer Register
Holds the location in main memory associated with the current procedure being performed. A procedure consists of some instructions and some temporary data.
SPR = Stack Pointer Register

Holds the location in the current procedure's scratch area where the next unused space is available. A subtle point is that the temporary data of the current procedure may either be stored in main memory or in some of the General Purpose Registers.

The notation SP(0) refers to the first unused word referenced by the SPR. SP(1) refers to the used word below SP(0) and so on.

Fundamental Instructions

Computers have a fixed set of built-in instructions, each of which has a specific representation determining, e.g. where it finds it's data, where it leaves any results, etc. When a computer program is being processed, the computer simply performs each instruction in the order they are loaded into memory, except for branch instructions which may store a new address into the PCR. The details of instructions vary from computer to computer, but here are a few typical examples:
Load, Store, Dup, Flip Instructions
LOAD
  • Before: SP(1) contains a main memory address divisible by 4.
  • After: SP(1) is loaded with the four bytes from main memory location Before: SP(1).
STORE
  • Before: SP(2) contains a main memory address divisible by 4.
  • Before: SP(1) contains some value.
  • After: The four bytes of Before:SP(1) are stored in main memory at Before:SP(2).
  • After: SPR has decremented by 8 bytes, we've popped two words off the stack.
DUP
  • Before: SP(1) contains some value.
  • After: SPR has incremented by 4 bytes.
  • SP(1) contains a copy of SP(2) which is the same as Before:SP(1).
FLIP
  • Before: SP(1) contains some value.
  • Before: SP(2) contains some value.
  • After: SP(1) = Before:SP(2).
  • After: SP(2) = Before:SP(1).
Arithmetic Instructions

Integer arithmetic is exact, e.g. 10 / 3 = 3 remainder 1.
Floating point arithmetic is inexact, e.g. 10.0 / 3.0 = 3.3333333 where the number of digits in the output is a function of the precision. On the other hand, integer arithmetic can only span a short range of values. A floating point value uses part of its bits for an exponent and the rest for a mantissa. The exponent holds the exact magnitude of the value, while the mantissa holds the approximate value. Scientists and engineers like floating point because their measurements are usually approximate anyway, but they may span very large magnitudes.

Most modern machines have simple integer arithetic instructions and very fancy floating-point instructions. For simplicity, our model computer handles floating-point arithmetic as much as possible like the integer arithmetic.

Input/Output Instructions

Input and Output involves transferring data to or from something outside the little world of our model computer. This data transfer requires coordinating the action of our general-purpose CPU with the special-purpose device processor attached to the device. Device addresses are unique numbers identifying a specific device.

Managing Input/Output efficiently requires some sophisticated software called a device driver. There are a number of subtle issues here which will be dealt with later!

SEND X=device-address Y=memory-location Z=num-bytes ZZ=bytes-sent
Requests that the device at bus location X receive up to Z bytes from memory location Y. The device (printer, ethernet channel, etc.) might not be able to accept that much data, so the number of bytes it was able to receive is stored in ZZ.
RECV X=device-address Y=memory-location Z=num-bytes ZZ=bytes-sent
Requests that the device at bus location X send up to Z bytes to memory location Y. The device (printer, ethernet channel, etc.) might have that much data available, so the number of bytes it was able to send is stored in ZZ.

Here's how the computer accesses data from main memory:

  1. An instruction places the address of a byte in main memory into the MAR.
  2. The computer issues a LOAD WORD instruction.
  3. This causes the four bytes (32-bits) in main memory starting at the address stored in the MAR to be transferred to the MCR, replacing the previous settings of those switches we call the MCR.

Storing data into main memory is very similar:

  1. An instruction places the address of a byte in main memory into the MAR.
  2. The computer issues a STORE WORD instruction.
  3. This causes the four bytes (32-bits) in the MCR to be transferred to main memory starting at the address stored in the MAR, replacing the previous settings of those switches in main memory.

Because we can access main memory in any order, simply by placing the address of the first byte we want to load or store into the MAR, we sometimes call main memory Random Access Memory or RAM.
