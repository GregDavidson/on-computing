Advanced Representation

Preface

Fundamentally computers store and manipulate bits, bytes and words. Please read the Simple Note What Computers Are to learn about such things.

How do we get computers to be represent and meaningfully process the more abstract kinds of information which human beings understand, such as money, melodies, haut cuisine, fair allocation of resources, strategies for quitting smoking, etc., etc., etc.?

Computer Scientists and hackers have devised ways to build higher-level knowledge representations out of lower-level representations, building a tower of abstractions. Out of bits, bytes and words we build simple objects and classes. Out of objects and classes we build knowledge structures and concepts. Sometimes it gets a bit messy!

Currently this Simple Note is just a list of some of these higher-level representations. The only way to really learn software abstraction is to write programs which use abstraction to address interesting problems.

Atomic Data Structures

Enumerations
An enumeration type is simply a list of named values, e.g.
  • enumeration eye_colors = {Black, Brown, Hazel, Green, Blue}
An enumeration value is simply a specific value of a particular type, e.g.
  • eye_colors my_eye_color=hazel
.
Numbers
Computers only directly understand a few sizes of integers and floating point numbers. Out of these basics we can create integers of unlimited size, rational numbers to represent fractions and floating point numbers of arbitrary range and precision. Then we can go on to create complex numbers, quaternions, vectors, matrices, exact real numbers and other high-level kinds of numbers. Some languages let us directly specify our desired range and precision, e.g.
  • integer subrange month_days = 1..31

Hierarchical and Linked Data Structures

Arrays
An array is an aggregation (bunch) of values called elements. The elements they must be of the same type. They are placed next to each other in the computer's memory. One can process all the items the same way, or select specific items for special attention by their index (position) in the array. One can also rearrange the items. More advanced array systems allow arrays to grow and shrink by adding or removing items. Consider the handy arrays
  • Normal_Month_Lengths = {31,28,31,30,31,30,31,31,30,31,30,31}
  • Weekday_Names = {"Sunday", "Monday", "Tuesday", "Wednesday", "Thursday", "Friday", "Saturday"}.
Records
Also called a structure, tuple (short formultituple) or row , a Record is an aggregation (bunch) of values called fields (or slots or columns). Each field is given a name. Different fields do not need to be of the same type. Fields are typically stored next to each other in the computer's memory, although sometimes with gaps to ensure that fields begin on word boundaries for efficiency. One can select specific fields items by their names.

Search

Every choice of representation implicitly defines a search space. Computation can be viewed as a search process which creates and examines specific states within a search space. The choice of representation and the manner in which one searches it determines the power and efficiency of the computation.
